Written for a true beginner: what a Raspberry Pi actually is, how to talk to it, then 26 builds — each with a priced shopping list, step-by-step instructions with checkpoints, the item settings that make it earn, and the safety rules. AWhat is a Raspberry Pi, actually?

A Raspberry Pi is a complete computer the size of a deck of cards, sold bare for $15–55. It runs Linux (a free operating system), connects to WiFi, and can run programs — same as a laptop with three differences that make it perfect for this project:

  • Its "hard drive" is a microSD card. You prepare the card on your Mac by copying a ready-made operating system onto it ("flashing an image"). Card goes in Pi, Pi boots. If you ever wreck the software, re-flash the card — the Pi itself is nearly unbrickable.
  • It has no screen and doesn't need one. You type commands to it from your Mac over WiFi using SSH ("secure shell" — think: your Terminal window becomes the Pi's keyboard and screen). Running screenless is called headless, and it's normal.
  • It has GPIO pins — the superpower. The double row of 40 metal pins along its edge are switches your programs control. A line of code turns pin 17 on: electricity flows. Off: it stops. Read a pin: was a button pressed? Every recipe in this book is ultimately "a phone press on your website flips a GPIO pin (or sends a network command), and the pin drives a light, motor, pump, or relay."

The mental model: the Pi is a computer whose fingers you can program. Your Volt server is the brain deciding who's allowed to press; the Pi is the hand in the room doing the pressing.

The terminal, in ten commands

The terminal is just a way of operating a computer by typing instead of clicking. When you SSH into the Pi, you'll see a prompt like will@volt-pi-1:~ $ — it's waiting for you to type a command and press Enter. These ten cover this entire book:

Type thisWhat it does, in plain words
lslist the files in the folder you're standing in
cd voltstep into the folder called volt (cd .. steps back out)
mkdir voltmake a new folder
nano file.mjsopen a simple text editor. Type/paste, then Ctrl-O, Enter to save, Ctrl-X to exit
sudo …"do this as administrator" — needed for installing software and system changes
sudo apt install -y mpdinstall a program from Linux's built-in free app store (apt)
node listen.mjsrun a JavaScript program. Ctrl-C stops a running program
scp song.mp3 will@volt-pi-1.local:~/volt/copy a file FROM your Mac TO the Pi (run this one on the Mac)
sudo systemctl restart volt-rigrestart your auto-start service (see §C step 8)
journalctl -u volt-rig -fwatch your service's live logs — your window into what the Pi is doing

Two habits that make everything easier: press Tab while typing a file/folder name and the terminal finishes it for you; press the up arrow to get your previous command back instead of retyping it.

Words you'll meet (one-line glossary)

GPIO
the Pi's 40 programmable pins. Numbered by "BCM" name (GPIO17), not by physical position — search "pi pinout" for the map.
Relay
an electrically-operated switch. Small signal from the Pi clicks a big switch for a separate circuit. How we "press buttons" on other machines.
Servo
a small motor that rotates to the exact angle you ask (0–180°) and holds it. Aiming, joints, levers.
Continuous servo / DC motor
spins like a wheel instead of holding an angle. Wheels, augers, spools.
Stepper
a motor that moves in exact tiny clicks — repeatable positioning for plotters and gantries.
Solenoid
an electromagnet that punches a rod out ~1 cm. Strikes and pokes.
MOSFET board
an electronic valve letting a tiny GPIO signal switch a big current (pumps, solenoids, motors). Like a relay but silent and fast.
PWM
switching a pin on/off very fast to express "how much" — how servos hear angles and motors hear speed.
PCA9685
a $12 helper board that drives up to 16 servos properly so the Pi doesn't have to. Used in every servo recipe.
I2C
a two-wire language the Pi uses to talk to helper boards like the PCA9685. Enable once in settings.
Common ground
every power supply's minus wire connected together. Electricity flows in loops; grounds join the loops. Rule #1 of wiring.
systemd service
Linux's "run this program automatically at boot and restart it if it dies." How rigs survive power cuts.

The five verbs — everything you will ever do with a Pi

  1. Flash it — copy the operating system onto the microSD (once).
  2. SSH in — open its terminal from your Mac.
  3. Install — sudo apt install … / npm i … the tools a recipe needs.
  4. Run — node yourscript.mjs and watch it work.
  5. Enshrine — wrap it in a systemd service so it runs forever without you.

Every recipe below is those five verbs plus wiring.

BOne-time Pi setup, click by click (~30 min)

  1. Flash the card. On your Mac, download Raspberry Pi Imager (raspberrypi.com/software). Insert the microSD via a USB reader. In Imager: Choose Device → your Pi model · Choose OS → Raspberry Pi OS (other) → Raspberry Pi OS Lite (64-bit) ("Lite" = no desktop; you won't need one) · Choose Storage → the card.
  2. The settings screen is the magic. Click Next → Edit Settings: set hostname volt-pi-1; a username + password (write them down!); your WiFi name + password exactly (capitals matter) with country US; and under Services: Enable SSH, "use password authentication." Save → Yes → wait → eject.
    Why: this is how the Pi boots already on your WiFi with SSH open — no monitor or keyboard ever needed.
  3. First boot. Card into the Pi, power in. Wait a full 2–3 minutes (first boot resizes things). The green LED flickers = it's alive.
  4. Connect. Mac Terminal: ssh will@volt-pi-1.local (your username, your hostname). Type yes to the first-time fingerprint question, then your password (typing shows nothing — that's normal, it's hiding it).
    ✅ Checkpoint: your prompt now says will@volt-pi-1:~ $ — you are typing ON the Pi.
  5. Update + install Node. One line at a time (a Zero takes a few minutes each): 
    sudo apt update && sudo apt full-upgrade -y
sudo apt install -y nodejs npm git
node -v
    ✅ Checkpoint: node -v prints v18 or higher. If lower: curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - && sudo apt install -y nodejs
  6. Project folder. mkdir ~/volt && cd ~/volt && npm init -y && npm i ws (ws = the WebSocket library — the Pi's phone line to your server).
  7. The listen test — proves everything with zero hardware. Create an item in your ops page and note its 6-char code. On the Pi: nano listen.mjs, paste the listener from VOLT-PI-PLAYBOOK.md §3.6, save, and run node listen.mjs YOURCODE. From your Mac, inject a fake press: 
    curl -X POST https://td-stream-control.onrender.com/api/channels/item:YOURCODE/actions \
  -H 'content-type: application/json' -H 'X-Admin-Key: YOUR_KEY' \
  -d '{"type":"key","action":"pad_up","user":{"name":"test"}}'
    ✅ Checkpoint: the Pi prints PRESS pad_up by test. Phone → server → Pi is proven; every recipe now just decides what a press DOES.
  8. Enshrine it. Create the auto-start service (playbook §3.7 has the file): sudo systemctl enable --now volt-rig. Pull the Pi's power, plug back in, and watch it come back on its own: journalctl -u volt-rig -f.
    After the outputs mission ships, your service runs the real bus-to-pi.mjs with a rigKey — and the ops page shows a green presence dot.

If stuck: can't find the Pi → it's not on WiFi; on a Zero 2 W the #1 cause is a 5 GHz-only network name (it speaks 2.4 GHz only) — or find its IP in your router's client list. Random crashes → cheap power supply. Password rejected → remember Imager's username, not "pi".

CWiring for people who've never wired

  • GPIO pins are weak on purpose. They can light an LED (with a 330 Ω resistor); anything hungrier — motor, pump, strip, solenoid — needs a driver in between (relay, MOSFET, PCA9685, motor board). The recipes always name the driver.
  • Common ground, always. Any extra power supply's minus wire must also connect to a Pi ground pin. Symptoms of forgetting: servos twitch, nothing works reliably.
  • Never power motors/servos from the Pi's 5 V pin. Separate supply (+ common ground) or the Pi browns out and corrupts its SD card.
  • Never touch mains (wall power). Wall-powered things get switched by a smart plug the Pi commands over WiFi, or a properly enclosed relay HAT. Full stop.
  • First circuit ritual: LED + resistor from GPIO17 to GND, blink it from a phone press. Do it once — everything else is this with bigger drivers.
The universal trick — a relay across a button. Anything with a physical button (RC remote, claw machine panel, bubble gun, arcade board, pinball flipper button) can be Volt-controlled without understanding its electronics: open it, find the two contacts the button connects, wire a relay's output terminals across them, and the Pi "presses the button" by clicking the relay. Find the contacts with a $15 multimeter in continuity (beep) mode: touch the probes to pairs of points while pressing the button — beep only while pressed = you found them. This trick unlocks a third of this book.

R0The Soundboard

EASY · ONE EVENING~$20 + speakerWORKS TODAY

Press A/B/C and airhorns, laughs, and beat-drops play through a speaker in the room.

What you're really building: the listen test from §B, plus one line that plays a sound file instead of printing. The "hardware" is a speaker cable — that's why it's Recipe 0.
Part~$Note
Kit A—Pi Zero has no headphone jack…
USB audio adapter$8…this adds one (or use a Pi 3/4's jack, or HDMI audio)
Powered speakerhave oneanything with a 3.5 mm input
  1. Finish §B through the listen test. Plug adapter into Pi, speaker into adapter, volume knob to ~half.
  2. Install players: sudo apt install -y mpg123 alsa-utils.
  3. On your Mac, gather 7 short MP3s (airhorn, drum, laugh…) and copy them over: scp *.mp3 will@volt-pi-1.local:~/volt/sounds/ (make the folder first on the Pi: mkdir ~/volt/sounds).
  4. Test by hand ON the Pi: mpg123 ~/volt/sounds/horn.mp3.
    ✅ Checkpoint: you hear it. Silent? Run alsamixer, press F6, choose the USB device, arrow keys to raise volume; also check the speaker's own knob.
  5. Open your listener (nano listen.mjs) and add at the top: 
    import { execFile } from 'node:child_process';
const SOUND = { pad_up:'drum.mp3', pad_down:'bass.mp3', pad_left:'laugh.mp3',
  pad_right:'record.mp3', btn_a:'horn.mp3', btn_b:'yeah.mp3', btn_c:'drop.mp3' };
    …and inside the branch that prints PRESS, add: 
    if (SOUND[m.action]) execFile('mpg123', ['-q', `${process.env.HOME}/volt/sounds/${SOUND[m.action]}`]);
  6. Run it, buy your own item slot from your phone, and mash buttons. ✅ Checkpoint: your thumbs are playing the room.
  7. Point the systemd service at this script (§B.8). Done — first earning item.
Earning settings: buy-now · $1 · 30 s · generous limits.

If stuck: sounds overlap into chaos → add a 300 ms cooldown per action in your map; wrong output device after reboot → sudo raspi-config → System → Audio → pick the USB device.

R1The Beacon

EASY · ONE EVENING~$12WORKS TODAY · ZERO WIRING

A lamp, neon sign, or rope light obeys the crowd: on, off, flicker, party.

What you're really building: the Pi sends tiny web requests to a smart plug on the same WiFi. No wires at all — the plug IS the safety barrier between you and wall power.
Part~$Note
Tasmota smart plug$12buy “Tasmota pre-flashed” — controlled locally by web requests, no cloud account
A light with characterhave oneneon, lava lamp, marquee bulb…
  1. Plug the smart plug in near your router first; join its setup WiFi per its leaflet and give it your venue WiFi's name + password.
  2. Find its IP address: your router's app/page lists connected devices — look for “tasmota-…”. Reserve that IP (routers call it “DHCP reservation”) so it never changes.
  3. Open a browser to http://PLUG-IP — Tasmota shows a toggle page. Click it. ✅ Checkpoint: the lamp clicks on/off from your browser.
  4. Now from the Pi: curl "http://PLUG-IP/cm?cmnd=Power%20TOGGLE" — same click, but a computer did it.
  5. Map actions in your listener with the R0 pattern (execFile curl): A = Power%20On, B = Power%20Off, C = a 5-toggle party burst (loop with 400 ms sleeps), pad_up/down = slow/fast flicker loops that STOP when another press arrives.
  6. Add a 500 ms cooldown per action — relays physically click; don't machine-gun them.
  7. Item + QR + systemd, as always.
Earning settings: buy-now · $1 · 60 s · cooldownMs ≥ 500.

If stuck: curl works from your Mac but not the Pi → they're on different networks (guest WiFi isolation); put Pi and plug on the same one.

R2The Light Wall

EASY · ONE EVENING~$30WORKS TODAY · BEST $/WOW

A whole wall, window, or bar-back of LEDs changes pattern, color, and speed under their thumbs.

What you're really building: a $8 ESP32 chip runs WLED (free software that's basically a lighting console); the Pi just tells it which preset to play. You solder nothing if you buy strips with connectors.
Part~$Note
WS2812B strip, 5 V, 5 m$18IP65 (coated) if near drinks; “with connectors”
ESP32 dev board$8runs WLED
5 V supply sized to strip$10~3 A per meter at full white; cap brightness and half it
Aluminum diffuser channel$10optional; turns LED dots into a pro glow
  1. Flash WLED from your Mac's Chrome: go to install.wled.me, plug the ESP32 in by USB, click Install. (This is the same "flash an image" idea as the Pi, just simpler.)
  2. Connect to the “WLED-AP” WiFi it creates → a setup page appears → give it the venue WiFi. Find + reserve its IP like R1 step 2.
  3. Wire three connections: strip 5V→supply+, strip GND→supply− and a wire from supply− to an ESP32 GND pin (common ground!), strip DIN (data)→ESP32 pin GPIO2.
  4. Open http://WLED-IP: set the LED count, set brightness limit ~60% (looks identical, halves heat), play with effects. ✅ Checkpoint: the strip animates from your phone's browser.
  5. Save 8 presets (calm / rainbow / chase / sparkle / strobe-lite / sunset / ocean / party) as Presets 1–8 in WLED's UI.
  6. From the Pi: curl "http://WLED-IP/win&PL=3" → preset 3 plays. Map: A/B/C = presets, pad_up/down = brightness (&A=64…&A=255), pad_left/right = effect speed (&SX=…).
  7. Mount in the diffuser channel where the street can see it. Item + QR + systemd.
Earning settings: buy-now · $1–2 · 60–90 s. Build this FIRST for a venue pitch — visible from outside, zero moving parts, nothing to break.

If stuck: first LED works, rest flicker → inject power at both ends of the strip (5 V + GND at the far end too); colors wrong → set strip type to WS2812B GRB in WLED settings.

R3The Searchlight

MEDIUM · ONE WEEKEND~$40

A spotlight physically swings where they steer — across the ceiling, the dance floor, the mural.

What you're really building: your first servo project. Two servos (pan + tilt) hold a light; each phone press nudges an angle. The PCA9685 board does the precise electrical timing so your code just says "go to 85°".
Part~$Note
Kit B$25servos + PCA9685 + 5 V 3 A supply
SG90 pan/tilt bracket kit$9screws together in 10 minutes
Small 12 V LED spot + MOSFET board$12or keep the light always-on and sell only the aim
  1. Enable the helper-board language: sudo raspi-config → Interface Options → I2C → Yes. Reboot (sudo reboot, SSH back in after 60 s).
  2. Wire the PCA9685 (4 wires to the Pi): VCC→3.3 V (pin 1), GND→GND (pin 6), SDA→GPIO2 (pin 3), SCL→GPIO3 (pin 5). Then servo power: the 5 V 3 A supply's + to the board's V+ screw terminal, its − to the board's GND and a Pi GND (common ground).
  3. Prove the Pi sees the board: sudo apt install -y i2c-tools && i2cdetect -y 1. ✅ Checkpoint: a grid prints with 40 in it.
  4. Plug servos into channels 0 (pan) and 1 (tilt) — orange wire = signal, toward the board's labels.
  5. Center before assembling: npm i pca9685 i2c-bus, write 6 lines to send both channels to 90°, THEN screw the bracket together with servos at center. (Skipping this is why brackets bind and servos strip.)
  6. Write the driver: keep two variables pan, tilt; each d-pad press (hold-repeat arrives ~8/s) nudges ±2°; clamp hard — e.g. pan 20–160°, tilt 30–90° — so it can never aim at eyes, windows, or the street. Move by easing 3°/frame toward the target: smooth reads expensive, jumpy reads broken.
  7. A = pulse the light (MOSFET on GPIO17: on 300 ms, off), B = recenter to 90/60.
  8. Deliberately test the corners: hold each direction until it stops at the clamp. ✅ Checkpoint: it physically cannot point anywhere unsafe.
Earning settings: buy-now · $2 · 60 s · maxPerMin so servos rest between slots.
Never a laser without real optics knowledge; never any beam clamped where a face can be. The clamp is the safety device — test it like one.

R4The Candy Drop

MEDIUM · ONE WEEKEND~$25

Press A → a scoop of candy rattles into the tray. A vending machine that cost $40.

What you're really building: a motorized auger (screw) at the bottom of a candy hopper. One press = one metered turn = one fair scoop. The whole game is calibrating "one turn".
Part~$Note
Continuous-rotation servo (FS90R)$8spins like a motor, wires like a servo (PCA9685 ch0)
Kit B (PCA9685 etc.)—from R3
Hopper: 2 L bottle + funnel + auger$83D-print an auger, or hot-glue a spiral of stiff tubing around a dowel on the servo horn
Wrapped candy$10wrapped = zero health-code drama
  1. Same I2C setup as R3 (done already if you built it). CR servo → channel 0.
  2. Learn its language: a CR servo treats "angle" as speed — 90° = stop, 130° = spin one way, 50° = the other. Find the exact stop value for YOUR servo first (they drift a few degrees) and write it down.
  3. Build the tube: a cardboard/PVC tube angled 30° down to the tray; the auger sits inside; the bottle feeds it from above through the funnel.
  4. Calibrate the dose: spin in 0.25 s bursts, count candies out. Tune burst length until one press ≈ 4–6 candies. ✅ Checkpoint: ten presses in a row give consistent doses.
  5. Add an anti-jam tick: after each dose, reverse 0.1 s. Candy bridges collapse.
  6. Map A = one dose; everything else can trigger R0 sounds (“ka-ching”).
  7. Tape the QR ON the hopper — people buy what they're staring at.
Earning settings: buy-now · price = candy cost × 4 · 15 s slot (a slot IS one vend) · huge cooldownMs so one purchase = one dose.

If stuck: jams → auger too tight in the tube (loosen fit) or candy too big (pick smaller candy; round beats flat).

R5The Drumbot

MEDIUM · ONE WEEKEND~$45

Solenoids strike a real snare, cowbell, and woodblock in time with their taps — the audience plays percussion on your radio stream.

What you're really building: three electric drumsticks. A solenoid is an electromagnet that punches a rod out ~1 cm when powered; a MOSFET board lets a weak GPIO pin fire that strong punch. Short pulses = strikes.
Part~$Note
3× 12 V push solenoids$18“push type, 10 mm stroke”
MOSFET boards ×3 (or one 4-ch)$8the muscle
1N4007 flyback diodes$2across each coil — absorbs the coil's kickback; skip = fried boards
12 V 2 A supply$10common ground with the Pi
Clamps + scrap wood$8mounts
  1. Wire ONE first: supply + → solenoid wire 1; solenoid wire 2 → MOSFET board's OUT+… follow the board's labels (V+ / V− / OUT / signal); signal → GPIO17; ALL grounds together. Diode across the solenoid wires, stripe (cathode) toward +12 V.
  2. Fire it from code: pin high 60 ms, then low. ✅ Checkpoint: one clean “thunk,” not a buzz (buzz = pulse too long).
  3. Clamp it ~5 mm from the drum head; tune the gap by ear — closer is tighter, farther is boomier.
  4. Repeat ×3 (GPIO 17/27/22 → snare/cowbell/woodblock).
  5. Map A/B/C = one strike each (40–80 ms), with 120 ms per-solenoid cooldown — coils overheat if machine-gunned; the cooldown is also your rhythm quantizer, weirdly musical.
  6. Mic it into the PA or your stream. The radio loop is the whole charm.
Earning settings: buy-now · $2 · 45 s · cooldown 120 ms · maxPerMin ~200.

If stuck: MOSFET board dies on the second strike → you skipped the diode; Pi reboots when solenoid fires → grounds not common or supply too weak.

R7The Claw Machine FLAGSHIP

ADVANCED · TWO WEEKENDS$200–450 all-in

A real claw machine, played from their phone. The best coin-op economics ever invented — minus the coin box.

What you're really building: the universal trick (§C) at scale. The machine already knows how to be a claw — motors, limits, grab strength, prize return. You wire five relays across its five buttons and the Pi becomes a very fast player pressing them.
Part~$Note
Used countertop claw machine$150–400Marketplace/OfferUp — “powers on, claw moves” is all it needs; ignore broken coin mechs
8-ch relay module$9your electric fingers (5 used)
Multimeter—Kit A
Prizes$30small plush, or shop-credit cards in capsules

R9The RC Arena

MEDIUM · ONE WEEKEND~$45

A little car in a walled tabletop arena — push the ball into the goal, knock the cones, beat the clock.

What you're really building: the button-hack on the car's REMOTE. The factory radio link keeps doing the hard part; the Pi just presses the remote's buttons with optocouplers (gentler relays for tiny electronics).
Part~$Note
Cheap RC car$25the $25 kind with a 4-button remote (fwd/back/left/right)
4× PC817 optocoupler (or 4-ch relay)$4solder across the remote's button pads
Lumber + plexi$15arena walls ~10 cm, one clear side
  1. Open the remote (4 screws). Each rubber button sits on two exposed pads on the board.
  2. Solder two thin wires to each button's pads (8 wires). Label them.

FRobotics — controlling real movement

Robotics here isn't humanoids — it's giving strangers safe, bounded actuation. Every motion needs a driver (what powers it), a mapping (what a thumb does to it), and an envelope (what it can never do). Get those three right and almost anything can be an item.

The movement vocabulary — what a d-pad + A/B/C can drive

Aim (rotation to an angle) — hobby servos, 0–180°.
Pan/tilt heads, joints, dials, mirrors, eyes. Map: d-pad nudges ±2° with hold-repeat; clamp ranges hard. → R3, R14, R18, R25
Spin (continuous rotation) — DC motors, CR servos.
Wheels, augers, spools, turntables. Map: hold-to-spin; timed bursts for metering. → R4, R16, R21
Step (precise increments) — stepper motors.
Plotters, gantries, patterns that must repeat exactly. Map: jog per press; home at boot. → R6, R7, R20
Push/strike (linear, fast) — solenoids.
Drums, pokes, latches, triggers. Map: one press = one timed pulse + cooldown. → R5, R15, R22
Reach (linear, slow & strong) — linear actuators, winches, lead screws.
Lift, reveal, open, tilt heavy things. Map: A = scripted travel between end-stops. → R17
Grip — a servo squeezing a printed claw.
Pick-and-place, handing things over. Map: C toggles open/closed. → R14
Drive (locomotion) — two motors, differential steering.
Rovers, floor bots, anything in a pen. Map: tank-mix the d-pad. → R16
Flow (fluid/air) — pumps, valves, fans, foggers.
Water, mist, bubbles, fog rings, drinks. Map: timed bursts, hard caps. → R11, R21, R23
The safety envelope (every robotics recipe): software limits on every axis, tested at the corners · hardware end-stop switches in SERIES with motor power on anything linear or strong · server-side duty limits sized to the actuator · a physical kill switch on the supply · speed-ease everything · and the rule that settles arguments: if it can pinch, poke an eye, or reach a face — change the design, not the warning sign.
